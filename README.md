# ЕГЭ Обществознание — интерактивный тренажёр

Веб-приложение для подготовки к ЕГЭ по обществознанию. Цель — **85+ баллов**. Полностью на русском языке, тёмная тема, адаптивная вёрстка.

🔗 **Демо (GitHub Pages):** https://realfactchecknews-eng.github.io/ege-obshestvoznanie/

## Возможности

- **📘 Теория** — материал по 6 разделам кодификатора ФИПИ (Человек и общество, Духовная культура, Экономика, Социальные отношения, Политика, Право). Раскрывающиеся карточки тем, ключевые термины с всплывающими определениями, отметка «изучено».
- **✎ Задания** — 60 реальных заданий в формате ЕГЭ. Часть 1 (№1–20): выбор одного/нескольких ответов, установление соответствия, вписывание слова. Часть 2 (№21–25): анализ текста, план, мини-сочинение — с эталонным ответом и **критериями оценивания ФИПИ**. После ответа — правильный вариант и пояснение со ссылками на статьи законов.
- **⏱ Вариант целиком** — сборка варианта из 25 заданий с таймером **3 ч 55 мин**, проверкой части 1 и разбивкой результата по разделам. История попыток сохраняется.
- **◔ Слабые места** — дашборд точности по темам и номерам заданий; темы ниже 60% выделяются красным; кнопка «тренировать слабые темы».
- **⚏ Карточки терминов** — режим интервального повторения (Leitner): термин → определение → «знал / не знал»; незнакомые карточки возвращаются раньше.
- **📊 Прогресс** — счётчик решённых за день, серия дней подряд (streak), общая статистика.

Прогресс хранится в `localStorage` — авторизация не требуется.

## Технологии

- **Клиент:** React + TypeScript + Vite + Tailwind CSS (статический SPA, хостится на GitHub Pages).
- **Сервер (опционально, для локального запуска):** Node.js + Express + SQLite (`better-sqlite3`).

> GitHub Pages раздаёт только статику, поэтому развёрнутая версия работает целиком на клиенте (данные заданий бандлятся в сборку, прогресс — в `localStorage`). Папка `/server` — полноценный REST API для тех, кто хочет хранить данные в SQLite.

## Структура

```
client/                 # React-приложение (то, что деплоится)
  src/
    data/               # tasks.json, theory.json, terms.json — источник данных
    components/          # TaskCard и др.
    pages/              # Dashboard, Theory, Practice, Variant, WeakSpots, Flashcards
    lib/                # data.ts (логика проверки), storage.ts (localStorage)
server/                 # Express + SQLite API
  data/                 # копии JSON
  db/seed.js            # заполнение базы
  index.js              # REST API
.github/workflows/      # автодеплой на GitHub Pages
```

## Запуск клиента (локально)

```bash
cd client
npm install
npm run dev      # http://localhost:5173/ege-obshestvoznanie/
npm run build    # production-сборка в client/dist
```

## Запуск сервера (опционально)

```bash
cd server
npm install
npm run seed     # создать и заполнить SQLite-базу из data/*.json
npm start        # API на http://localhost:3001
```

### Эндпоинты API

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/api/tasks?topic=&task_number=&limit=` | список заданий с фильтрами |
| GET | `/api/tasks/variant` | случайный вариант (по одному заданию на номер) |
| POST | `/api/progress` | сохранить результат ответа |
| GET | `/api/progress/stats` | статистика по темам и номерам |
| GET | `/api/theory` | теория по разделам |
| GET | `/api/terms` | словарь терминов |

## Как добавить задания

Откройте `client/src/data/tasks.json` и добавьте объект:

```json
{
  "id": 61,
  "year": 2024,
  "task_number": 6,
  "type": "multiple",
  "topic": "Экономика",
  "subtopic": "Рыночная экономика",
  "question": "Текст задания...",
  "options": ["вариант 1", "вариант 2", "вариант 3"],
  "correct_answer": "13",
  "explanation": "Пояснение со ссылкой на статью закона...",
  "source": "ФИПИ 2024"
}
```

Поддерживаемые `type`: `single`, `multiple`, `matching` (нужны `match_left`/`match_right`), `fill`, `written` (для части 2 — поля `criteria` и эталон в `correct_answer`). После правки JSON пересоберите клиент (`npm run build`) — изменения попадут и в сервер после `npm run seed`.

## Деплой

Пуш в ветку `main` автоматически собирает и публикует клиент на GitHub Pages через GitHub Actions (`.github/workflows/deploy.yml`).
